# Bangkok Seafood Price Advisor — เคสทดสอบ (ฉบับภาษาไทย)

**กลุ่มผู้อ่าน:** ทีมธุรกิจ / QA
**ระบบที่ทดสอบ:** LangGraph ReAct agent (Azure OpenAI `gpt-4o`) พร้อม reasoning layer + 6 เครื่องมือ
**วันที่รัน:** 2026-05-10
**โหมดการรัน:** เรียก agent แบบ live ผ่าน `tests/run_test_cases.py` (ใช้ flow `build_graph()` เดียวกับหน้าแชท)

> เอกสารนี้ใช้ **คำถามภาษาไทยจริง** (รันสด ไม่ใช่การแปลคำตอบ EN) เคสที่มีรหัสลงท้าย `-TH` เป็นเคส EN ที่แปลเป็นไทยแล้ว rerun ใหม่ ส่วน TC04 / TC09 / TC11 เป็นภาษาไทยอยู่แล้วตั้งแต่ชุดเดิม

## วิธีอ่านเอกสาร

แต่ละเคสประกอบด้วย: คำถามที่ผู้ใช้พิมพ์ → สิ่งที่ agent ควรทำ → สิ่งที่ agent ทำจริง → ผลตัดสิน Pass/Fail

- **Single tool** — คำถามเดียว → เครื่องมือเดียว ใช้ตรวจการ routing
- **Multi-step** — คำถามเดียว → เรียกเครื่องมือ 2 ตัวขึ้นไปต่อกัน
- **Edge case** — สินค้าที่ไม่มีในฐานข้อมูล, คำถามคลุมเครือ, หรือคำตอบต่อจาก clarification

เคสจะ **PASS** เมื่อคำตอบที่ผู้ใช้เห็น **ถูกต้องและเป็นประโยชน์** — ไม่จำเป็นต้องเรียกเครื่องมือตรงตามแผนเป๊ะ ๆ Agent อาจใช้เครื่องมืออื่นที่ให้ผลเทียบเท่า

| **ข้อสังเกตสำหรับนักพัฒนา** | Reasoning layer (`agent/reason.py`) ไม่ได้รับ instruction "ตอบในภาษาเดียวกับผู้ใช้" — มีเฉพาะใน `agent/prompts/system.py` ที่ใช้กับ agent_node เท่านั้น TC14-TH ใช้คำถามเดียวกันแต่ดันถามเป็นไทย แสดงว่าเป็นพฤติกรรมไม่คงที่ ควรเพิ่มกฎภาษาเข้า REASON_SYSTEM_PROMPT |

## ข้อสังเกตหลังรันชุด TH

1. **ภาษาคำตอบ**: เกือบทุกเคสตอบเป็นภาษาไทยตรงตามภาษาคำถาม สอดคล้องกฎ "respond in same language" ที่ตั้งไว้ใน `agent/prompts/system.py`
2. **TC08-TH**: Agent วิเคราะห์ว่าผู้ใช้ขอ briefing ภาษาไทย และส่ง `language="th"` ตอน call tool ได้ถูกต้อง — language argument routing ทำงาน
3. **TC13-TH (ข้อสังเกต)**: Reasoning layer (`agent/reason.py`) ไม่มีกฎเรื่องภาษาในการ clarification ส่งผลให้บางครั้งถามเป็นอังกฤษทั้งที่ผู้ใช้พิมพ์ไทย เห็นได้จาก TC13-TH (EN clarification) vs TC14-TH (TH clarification) ที่ใช้คำถามเดียวกัน แต่ผลต่างกัน ควรเพิ่ม "ตอบ clarification ในภาษาเดียวกับผู้ใช้" เข้า `REASON_SYSTEM_PROMPT` ในรอบถัดไป
4. **การสลับเครื่องมือ** (TC05-TH, TC10-TH): พฤติกรรมเหมือนชุด EN — Agent ใช้ `query_seafood_prices` แทน `get_price_trend` คำตอบยังถูกต้องในแง่ผู้ใช้
5. **Edge case ภาษาไทย** (TC12-TH): Agent ตอบสุภาพ ไม่กุข้อมูล และเสนอทางเลือก แม้ใช้คำว่า "กุ้งล็อบสเตอร์" ที่ไม่ตรงกับ category ใด ๆ
